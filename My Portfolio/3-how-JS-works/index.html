Section 3: How Javascript Works Behind the Scenes

Execution Context:

A box, a container or a wrapper which stores variables in which a piece of our code is evaluated and executed

Global Execution Context

  • Code is not inside any function
  • Associated with the Global Object
  • In the browser, that's the window object
lastName === window.lastName // true

The Variable Object

  • The argument object is created, containing all the arguments that were passed into the function.

    • Hoisting:

  • Code is scanned for cuntion declarations: for each function, a property is created in the Variable Object, pointing to the function.
  • Code is scanned for variable declarations: for each variable, a property is created in the Variable Object, and set to undefined.

Difference between functions and variables: Functions are already defined before the execution starts

Scoping in JavaScript

  • Scoping answers the question "where can we access a certain variable?"
  • Each new function creates a scope: the space/environment, in which the variables it defines are accessible.
  • Lexical Scoping: a function that is lexically within another function gets access to the scope of the outer function.

The 'THIS' Variable

  • Regular Function Call: The this keyword points at a global object (the window object, in the broswer).
  • Method Call: The this variable points to the object that is calling the method.
  • The this keyword is not assigned a value until a function where it is defined is actually called.
